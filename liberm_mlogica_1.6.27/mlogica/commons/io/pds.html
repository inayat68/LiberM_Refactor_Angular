    Partitioned Datasets

    LIBER*M Commons supports partitioned datasets (PDS) and maps them to file system directories, which children files are mapped to PDS members. The children file name must match the member name, without extension.

    For example, a PDS named ACME.PDSFILE.LIB and containing the members FILETXT1, COPY0001 and COPY0003, matches the following directory structure:

    Text Only
    $DATA_ROOT
    |
    + ACME.PDSFILE.LIB
            |
            +- FILETXT1
            +- COPY0001
            +- COPY0003

    PDS can be defined using nibvsam. They can be accessed passing them on the DD statement, using the parameter member to indicate, indeed, the desired PDS member.

    Example:

    Groovy
    def PDS = "TEST.PDS.FB80"
def pdsDir = "${System.env['DATA_ROOT']}/${PDS}"            // physical path interpolated for file/dir operation hereafter 
def pdsPhysical = "\${env:DATA_ROOT:-.}/${PDS}"             // physical path correctly escaped for the catalog
def path = new File(pdsDir)

NibVsamCommandLine.invoke(new String[]{"delete", PDS})
NibVsamCommandLine.invoke(new String[]{"define", "nonvsam", "--recfmt=LSEQ", "--lrecl=80,80", "--pds", "--physical=${pdsPhysical}", PDS})
path.mkdir()

exec "STPWRITE", pgm: "WRITFILE", parm: "WRITE", {
    dd "OUTFILE1", dsn: PDS, member: "MEMBER01", disp: [NEW, KEEP, KEEP]
}

exec "STPREAD1", pgm: "READFILE", parm: "READ", {
    dd "INFILE1", dsn: PDS, member: "MEMBER01", disp: [OLD, KEEP, KEEP]
}

    EBCDIC storage

    PDSes are normally used to store data cards, scripts and other "code". When LIBER*M Commons is configured to store data in EBCDIC mode, also PDS are supposed to be stored in EBCDIC. This may be unhandy in all those cases mentioned before, when users and/or developers need to edit the PDS members.

    In this case, when referencing a PDS member in a DD, LIBER*M Commons looks first for a file with a special extension (txt by default):

    • if found, it converts the files from ASCII to EBCDIC before passing to the program
    • otherwise, it passes the files to the program AS-IS

    This means you can mix EBCDIC and ASCII members at your convenience:

    • ASCII files with the ascii extension
    • EBCDIC files, with no extension

    The ascii extension is .txt by default but can be configured in the LIBER*M Commons configuration file.
