    Sequential files

    LIBER*M Commons provide the class SequentialFile for sequential file I/O. This class relies on plain O.S. sequential files and supports both PS and PO type datasets.

    It supports:

    • fixed length records (FB)
    • variable length records (VB, in both IBM and MicroFocus formats)
    • line sequential records (LSEQ)

    File Definition

    SequentialFile provides fluent API to create and define a file object. This increases the readability and helps former mainframe programmers to familiarize with the target environments.

    Like in COBOL, when you declare a sequential file, you need to indicate:

    • the external file name (like on the COBOL ASSIGN) that must match a DD name
    • one or more file records
    • an optional file status field

    Example

    Java
    void files() {
    final NChar type1Record = new NChar(80);
    final NChar type2Record = new NChar(80);
    final NZoned fs = new NZoned(2);

    SequentialFile outFile = new SequentialFile(context, "FBFILE")
            .recordIs(type1Record)
            .recordIs(type2Record)
            .fileStatusIs(fs);
}

    File Access

    Once the file is defined, it can be accessed using familiar I/O API such as open, readNext, write and close.

    Please note that the match between internal name and DD name is performed at open time.

    Java
    void files() {
    final NChar inRec = new NChar(80);
    final NZoned fs = new NZoned(2);

    SequentialFile inFile = new SequentialFile(context, "FBFILE")
            .recordIs(inRec)
            .fileStatusIs(fs);

    outFile.open(OpenMode.Input);

    outFile.write();
    outFile.close();
    //...
    inFile.open(OpenMode.Input);
    inFile.readNext();
    inFile.close();
}

    Error management

    SequentialFile offers different mechanisms to check and manage I/O responses. These mechanisms can be combined.

    File Status

    This is the mechanism that mimics the COBOL file management. If the file is declared with a 2 bytes file status field, the latter can be used after every I/O operation to check the status of the operation.

    Java
    void files() throws IOException {
    final NChar inRec = new NChar(80);
    final NZoned fs = new NZoned(2);

    SequentialFile inFile = new SequentialFile(context, "FBFILE")
            .recordIs(inRec)
            .fileStatusIs(fs);

    inFile.open(OpenMode.Input);
    if (!fs.isZero()) {
        throw new RuntimeException("Error opening file");
    }
    while (true) {
        inFile.readNext();
        switch (fs.getInt()) {
            case 10:            // end of file detected
                inFile.close();
                return;
            case 0:             // record correctly read
                processRecord();
                break;
            default:            // error detected   
                throw new IOException("error reading file FS=" + fs);
        }
    }
}

    LIBER*M Commons file classes support the same file status as COBOL. See File Status.

    I/O response

    Every I/O returns a FileIOResponse object, which provides methods to verify the operation result.

    Java
    void files() throws IOException {
    var resp = inFile.open(OpenMode.Input);
    if (!resp.isSuccess()) {
        throw new IOException("Error opening file");
    } else {
        System.out.println("File successfully open");
    }
    while (resp.isSuccess()) {
        resp = inFile.readNext();
        if (resp.isEof()) break;
        if (resp.isSuccess()) {
            processRecord();
        } else {
            throw new IOException("error reading file FS=" + resp.getFs());
        }
    }
    inFile.close();
}

    FileIOResponse also provide fluent methods activate callbacks on specific conditions, somehow like COBOL clauses such as AT END or INVALID KEY.

    Java
    void files() throws IOException {
    var resp = inFile.open(OpenMode.Input)
            .onSuccess(() -> {
                System.out.println("File successfully open");
            })
            .onFailure(() -> {
                throw new IOException("Error opening file");
            });
    while (resp.isSuccess()) {
        boolean eof = false;
        resp = inFile.readNext()
                .onSuccess(() -> {
                    processRecord();
                })
                .atEnd(() -> {
                    eof = true;
                })
                .onFailure(() -> {
                    if (!eof) throw new IOException("error reading file FS=" + fs);
                });
    }
}

    Exception handling

    By default, file classes do not throw exceptions when an error is detected during an I/O operation. This feature can be turned at object creation time, turning on exceptions management with the method exceptionsEnabled().

    Java
    void files() throws IOException {
    SequentialFile inFile = new SequentialFile(context, "FBFILE")
            .recordIs(inRec)
            .exceptionEnabled()                 // turning on exceptions throwing
            .fileStatusIs(fs);
    try {
        inFile.open(OpenMode.Input);
        while (true) {
            inFile.readNext();
            processRecord();
        }
    } catch (EndOfFileException eof) {
        // EOF, nothing to do
    } catch (FileIOException e) {
        throw new IOException("error reading file FS=" + e.getStatus());
    } finally {
        if (inFile.isOpen()) {
            inFile.close();
        }
    }
}
