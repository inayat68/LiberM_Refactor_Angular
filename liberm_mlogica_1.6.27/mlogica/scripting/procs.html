    PROCs

    LIBER*M supports both:

    • instream PROCs, defined inline as Groovy closures
    • external PROCs, implemented as Groovy scripts based on NibProc

    This matches the two most common migration patterns:

    • keeping a PROC definition near the invoking job
    • moving a reusable PROC to a separate script artifact

    Instream PROCs

    Instream PROCs are defined as Groovy closures and passed as the proc argument to exec.

    The closure may declare a parameter that receives the arguments passed by the invoking step.

    Example:

    Groovy
    def TESTPRC1 = {
    def args ->
        {
            proc "TESTPRC1"

            def op = args?.op ?: "ABEND"
            def outdsn = "GROOVY.TEST.PRC.FB80"

            exec "DELDEF1", pgm: "IEFBR14", {
                dd "DD1", dsn: outdsn, disp: [MOD, DELETE, DELETE]
            }
            exec "TESTHELP", pgm: "TESTHELP", parm: op, {
                dd "OUTFILE1", dsn: outdsn, disp: [NEW, KEEP, KEEP], lrecl: 80, recfm: FB
            }
        }
}

exec "STEP0001", proc: TESTPRC1, op: "WRITE", {
    dd "DELDEF1.DD1", dsn: "TEST.OVR.PRC", disp: [MOD, DELETE, DELETE]
    dd "TESTHELP.OUTFILE1", dsn: "TEST.OVR.PRC", disp: [NEW, KEEP, KEEP], lrecl: 80, recfm: FB
}

    How instream PROC arguments work

    Arguments passed to the exec statement are made available to the PROC closure through the closure parameter, for example args?.op.

    This is the standard way to model symbolic PROC parameters when translating a JCL PROC into an inline Groovy closure.

    PDS PROCs

    External PROCs, conceptually equivalent to PDS PROCs, are regular Groovy scripts based on NibProc.

    In this model, PROC parameters are exposed as script fields.

    DD overrides are provided using qualified names in the form STEP.DDNAME, just like in JCL override syntax.

    Example:

    Groovy
    package procs

import com.nib.groovybatch.NibProc
import com.nib.commons.io.tools.NibVsamCommandLine
import groovy.transform.BaseScript
import groovy.transform.Field

@BaseScript NibProc REPROC

// REPROC PROC                                                                   
@Field def CNTLLIB = ""

proc "REPROC"

// //PRC001 EXEC PGM=IDCAMS                                                        
// //SYSPRINT DD SYSOUT=*                                                          
// //FILEIN  DD DISP=SHR,                                                          
// //        DSN=NULLFILE                                                          
// //FILEOUT DD DISP=SHR,                                                          
// //        DSN=NULLFILE                                                          
// //SYSIN   DD DISP=SHR,                                                          
// //        DSN=&CNTLLIB(REPROCT)                                                 

exec "PRC001", pgm: "IDCAMS", {
    dd "SYSPRINT", sysout: "*"
    dd "FILEIN", dsn: "NULLFILE", disp: SHR
    dd "FILEOUT", dsn: "NULLFILE", disp: SHR
    dd "SYSIN", dsn: "${CNTLLIB}", member: "REPROCT", disp: SHR
}

    Example (invocation):

    Groovy
    // * *******************************************************************        
// * Repro the processed transaction file                                       
// * *******************************************************************         
// //STEP05R EXEC PROC=REPROC,                                                     
// // CNTLLIB=AWS.M2.CARDDEMO.CNTL                                                 
// //*                                                                             
// //PRC001.FILEIN  DD DISP=SHR,                                                   
// //        DSN=AWS.M2.CARDDEMO.TRANSACT.VSAM.KSDS                                
// //*                                                                             
// //PRC001.FILEOUT DD DISP=(NEW,CATLG,DELETE),                                    
// //        UNIT=SYSDA,                                                           
// //        DCB=(LRECL=350,RECFM=FB,BLKSIZE=0),                                   
// //        SPACE=(CYL,(1,1),RLSE),                                               
// //        DSN=AWS.M2.CARDDEMO.TRANSACT.BKUP(+1)                                 

exec "STEP05R", proc: REPROC, CNTLLIB: "AWS.M2.CARDDEMO.CNTL", {
    dd "PRC001.FILEIN", dsn: "AWS.M2.CARDDEMO.TRANSACT.VSAM.KSDS", disp: SHR
    dd "PRC001.FILEOUT", dsn: "AWS.M2.CARDDEMO.TRANSACT.BKUP", gdg: 1, disp: [NEW, CATLG, DELETE], lrecl: 350, recfm: FB
}

    Runtime behavior of external PROCs

    At runtime, NibJob instantiates the PROC script class, injects the declared parameters, shares the current job, and executes the PROC in the same job context.

    If the invoking step passes a parameter that is not declared in the PROC script, the runtime raises an error.

    Choosing between instream and external PROCs

    Use an instream PROC when:

    • the logic is local to one job
    • readability is better when the PROC stays close to the invoking steps
    • you are translating a short JCL PROC with limited reuse

    Use an external PROC when:

    • the PROC is reused by multiple jobs
    • the logic is large enough to deserve a dedicated script
    • you want a clearer separation between orchestration and reusable batch logic

    Migration notes

    When converting JCL PROCs to LIBER*M scripts, keep the following rules in mind:

    • symbolic PROC parameters typically become closure arguments or @Field values
    • PROC step overrides must use qualified DD names such as PRC001.SYSIN
    • the PROC itself does not create a separate job; it executes inside the current job context
    • NibProc is a specialization of NibJob, so PROC scripts use the same DSL primitives (proc, exec, dd)
