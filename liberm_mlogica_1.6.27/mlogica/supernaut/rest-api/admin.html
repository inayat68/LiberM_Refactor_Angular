    Admin API

    Endpoint

    • URL: /admin
    • Method: POST

    Request Body

    • Type: AdminRequest
      • Fields:
        • userid (string): The user ID for authentication.
        • password (string): The password for authentication.
        • command (enum): The command to execute. Supported values:
          • Ping: Checks the status of the region.
          • Status: Returns the core parameters of the region.
          • ShutdownRegion: Requests a shutdown of the region.
            • ShutdownNode: Requests a shutdown of a node.
            • ReloadClassloader: Reloads the classloader.
            • Set: Updates a region resource
            • Get: Retrieves a region resource
            • List: Lists region resources
          • file (FileArg): The file resource to be managed (optional, used for file operations).
          • program (ProgramArg): The program resource to be executed (optional, used for program operations).
          • tdqueue (TdQueueArg): The TD queue resource to be managed (optional, used for TD queue operations).
          • terminal (TerminalArg): The terminal resource to be managed (optional, used for terminal operations).
          • doctemplate (DoctemplateArg): The document template resource to be managed (optional, used for document template operations).
          • sessions (SessionArg): The active terminal sessions to be managed (optional, used for terminal sessions operations).

    Response

    • Type: AdminResponse
      • Fields:
        • response (enum): The status of the command. Possible values:
          • Success: The command was executed successfully.
          • Error: An error occurred while executing the command.
          • Unsupported: The command is not supported.
        • message (string): response message providing additional information about the command execution.
        • payload (object): command result, if applicable.

    Description

    The Admin API allows clients to perform administrative tasks on the region, such as checking its status, shutting it down, reloading the classloader, or managing resource. The API authenticates the user and processes the request based on the provided command.

    Security

    The admin command is handled through the transaction CADM. The command can be secured assigning the correct authorization groups to the transaction CADM.

    Commands

    The Admin API supports several commands that can be executed by authenticated users. Each command performs a specific action within the region. The commands include:

    Ping

    The command Ping checks the status of the region. Returns a success response if the region is operational.

    Status

    The command Status checks the status of the region and returns core parameter. Returns a success response and a map of parameters.

    Shutdown Region

    The command ShutdownRegion requests a shutdown of the entire region. This command initiates a graceful shutdown process, allowing the region to complete any ongoing tasks before shutting down.

    Shutdown Node

    The command ShutdownNode requests a shutdown of a specific node within the region. This is useful for managing the region's resources and ensuring that specific nodes can be taken offline without affecting the entire region.

    Reload Classloader

    The command ReloadClassloader reloads the classloader for the region. This is particularly useful for applying updates or changes to the application's classes and mapset without requiring a full or rolling restart of the region.

    Set, Get, List

    The commands Set, Get, and List are used to manage region resources, such as FCT, PCT, DCT, TCT and DOCTEMPLATE entries. The Set command updates a specific resource, the Get command retrieves a specific resource, and the List command lists resources.

    The command resource and parameters are provided in resource-specific arguments, such as file, program, tdqueue, terminal, and doctemplate. These arguments allow for detailed management of various resources within the region.

    Set and Get require the resource identifier to be specified on the field name. List does not require any parameter.

    The response payload contains the details of the resources being managed, including their attributes and status.

    File resources

    File resources are managed using the file argument, a FileArg object that supports the following properties:

    Property type Description
    name string The name of the file.
    open boolean Indicates if the file is open.
    enabled boolean Indicates if the file is enabled.
    addable boolean Indicates if the file is addable.
    browseable boolean Indicates if the file is browseable.
    deletable boolean Indicates if the file is deletable.
    readable boolean Indicates if the file is readable.
    updatable boolean Indicates if the file is updatable.

    Example:

    JSON
    {
  "userid": "admin",
  "password": "supersecret",
  "command": "Set",
  "file": {
    "name": "DATASET1",
    "open": true,
    "addable": false,
    "deletable": false,
    "updatable": false
  }
}

    Program resources

    Program resources are managed using the program argument, a ProgramArg object that supports the following arguments:

    Property type Description
    name string The name of the program.

    TD queue resources

    TD queue resources are managed using the tdqueue argument, a TdQueueArg object that supports the following arguments:

    Property type Description
    name string The name of the TD queue.
    open boolean Indicates if the TD queue is open.
    enabled boolean Indicates if the TD queue is enabled.
    triggerLevel integer Trigger level of the queue TD queue is enabled.

    Terminal resources

    TD queue resources are managed using the terminal argument, a TerminalArg object that supports the following arguments:

    Property type Description
    name string The name of the terminal (EIBTRMID).

    DOCTEMPLATE resources

    TD queue resources are managed using the doctemplate argument, a DocTemplateArg object that supports the following arguments:

    Property type Description
    name string The name of the DOCTEMPLATE.

    Active terminal sessions

    Active sessions are managed using the sessions argument, a SessionArg object that supports the following arguments:

    Property type Description
    name string The name of the terminal (EIBTRMID).

    Examples

    Setting a file as closed:

    JSON
    {
  "userid": "admin",
  "password": "supersecret",
  "command": "Set",
  "file": {
    "name": "DATASET1",
    "open": false
  }
}

    Getting a TD queue:

    JSON
    {
  "userid": "admin",
  "password": "supersecret",
  "command": "Get",
  "tdqueue": {
    "name": "TDI1"
  }
}

    Listing all terminals:

    JSON
    {
  "userid": "admin",
  "password": "supersecret",
  "command": "List",
  "terminal": {}
}

    Listing active sessions:

    JSON
    {
  "userid": "admin",
  "password": "supersecret",
  "command": "List",
  "sessions": {}
}

    cURL Example

    Bash
    curl -X POST http://yourapiurl.com/admin \
-H "Content-Type: application/json" \
-d '{
  "command": "ReloadClassloader",
  "userid": "adminUser",
  "password": "adminPassword",
}'
